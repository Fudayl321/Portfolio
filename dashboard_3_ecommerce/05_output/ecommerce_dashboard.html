Data Source — Kaggle (Olist)

Real Brazilian e-commerce orders Oct 2016–Aug 2018 · 8,000 records · Categories: electronics, clothing, home, sports, beauty, books & more · Includes reviews, payments, delivery data, and customer state

2

Issues Found & Fixed

15 negative prices — invalid entries → removed
10 delivery_days = 0 — physically impossible → set to NA, imputed
80 price NAs — imputed with category median (preserves category-level pricing)
40 delivery_days NAs — imputed with category median
40 review_score NAs — imputed with overall median (neutral assumption)

3

Feature Engineering

Derived: year_month (for trend grouping) · year/month/quarter · freight_pct (freight as % of order) · high_value flag (top 25th percentile). All date parsing handled with lubridate.

Statistical Tests Applied

Independent t-test — Do high-value orders take longer to deliver?
Result: t=-0.78, p=0.4364, mean diff=0.2 days
Spearman correlation — price vs review score: r = 0

R Code Snippets

Cleaning Pipeline (dplyr)

# Impute price with category median price_meds <- df %>% filter(!is.na(price)) %>% group_by(product_category) %>% summarise(med = median(price)) df <- df %>% left_join(price_meds, by = "product_category") %>% mutate(price = ifelse( is.na(price), med, price)) %>% select(-med)
